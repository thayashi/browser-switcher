Use it to preserve project context that is too detailed for `README.md` but important for future development: current behavior, architecture decisions, Windows/Tauri caveats, release/versioning notes, verification history, and known platform issues.

## Current State

Last updated: 2026-05-03

- Current app version: `0.1.4`.
- Primary branch: `main`.
- Feature/docs work should happen on a branch, not directly on `main`.
- Browser Switcher is a Windows tray app built with Tauri 2 and Rust.
- The tray menu lists installed browsers discovered from Windows registration data.
- Selecting a browser opens Windows Default Apps settings through `ms-settings:` URIs.
- Browser Switcher does not change default apps directly.
- The About window is a Tauri WebView showing static HTML from `dist/index.html`.
- The About window title bar and frame are native Windows UI.
- `index.html` is the source template; `scripts/build-about.mjs` generates `dist/index.html` and copies `dist/icon.png`.
- `npm run dev:about` exists for About window layout checks.

## Release And Versioning Notes

Last updated: 2026-05-03

When creating a build intended for install or GitHub distribution, bump the app version first. Windows Installer, Start menu shortcuts, executable metadata, and Windows icon/notification caches may not refresh reliably when a new installer reuses the same version.

Version must stay aligned in:

```text
package.json
package-lock.json
src-tauri/Cargo.toml
src-tauri/Cargo.lock
src-tauri/tauri.conf.json
dist/index.html
```

`dist/index.html` is generated by `npm run build:about`, which reads `package.json`.

The latest release build generated during development was:

```text
Browser Switcher_0.1.4_x64_en-US.msi
Browser Switcher_0.1.4_x64-setup.exe
```

Before installing a new build, fully quit any running `browser-switcher.exe` process.

## About Window Notes

Last updated: 2026-05-03

The About window is HTML inside a Tauri WebView, not a native dialog. The window frame is native.

The OK button must hide the current Tauri window through:

```js
window.__TAURI__.window.getCurrentWindow().hide();
```

Do not use `window.close()` for the OK button. In this app, Rust intercepts close requests to hide the About window instead of quitting the tray process. Calling `window.close()` from the WebView can leave the native window visible while the WebView navigates to a blank/error surface.

For layout checks, use:

```powershell
npm run dev:about
```

This sets `BROWSER_SWITCHER_SHOW_ABOUT_ON_START=1` for the dev run and shows the About window immediately. Do not rely on manually launching `target\debug\browser-switcher.exe` during `tauri dev` to force the singleton callback; that was observed to produce a WebView error page in development.

## Singleton And Tray Notes

Last updated: 2026-05-03

Multiple instances originally produced multiple tray processes/icons.

Current behavior:

- `tauri-plugin-single-instance` prevents duplicate processes from remaining.
- Duplicate launch refreshes/re-registers the tray icon.
- Duplicate launch shows/focuses the About window.

This matters because after singleton support was added, launching the app again no longer rebuilt the tray icon by creating a new process. The existing process must explicitly refresh the tray icon.

Relevant flow:

```text
duplicate launch -> refresh_tray_icon(app) -> show_about(app)
```

## Icon Notes

Last updated: 2026-05-03

The selected app icon source is under:

```text
icon-designs/ver2/icons-light/
```

Applied app icon assets live under:

```text
src-tauri/icons/
```

Important Windows packaging note: `src-tauri/icons/icon.ico` should be a Windows-compatible uncompressed DIB/BMP multi-size ICO. PNG-compressed ICO entries looked valid as a file but produced a bad/default icon when embedded into the Windows executable resource.

Windows icon caches can be stale. If Start menu, Startup Apps, or notification settings show old icons after installing a newer build, fully quit the app and consider restarting Explorer or Windows before treating it as an app bug.

## Browser Discovery Notes

Last updated: 2026-05-01

Browser discovery checks Windows registration data rather than a hardcoded browser list. Entries are treated as browsers only when registration capabilities include both `http` and `https` URL associations.

Firefox appeared twice because Windows registered it through both `RegisteredApplications` and `Clients\StartMenuInternet`. Deduplication now keys on normalized `registry_id`, allowing different display names for the same registration to merge.

Arc did not expose a standard executable path suitable for icon extraction in normal registration data. A deeper MSIX/AppModel lookup was explored and removed because it was too specific and brittle. The current behavior is to use the default browser fallback icon when a standard icon source is unavailable.
