# AGENTS.md

Guidance for Codex and other development agents working in this repository.

## Branch Policy

- The primary branch is `main`.
- Do not commit directly to `main`.
- Create a feature branch before making changes.
- Prefer branch names that describe the task, for example `codex-documentation-cleanup`.

## Project Shape

- This is a Windows-focused Tauri 2 app.
- Rust code lives under `src-tauri/src/`.
- The About window source template is `src-about/index.html`.
- `dist/index.html` and `dist/icon.png` are generated by `npm run build:about`.
- The About window is HTML in a Tauri WebView. The title bar and frame are native Windows UI.
- `docs/development-summary.md` is the living project context document.
- `docs/superpowers/` is intentionally kept for discussion, specs, and implementation plans.

## Development Commands

Run from PowerShell:

```powershell
cd C:\Users\thaya\LocalDocs\dev\browser-switcher
$env:PATH="$env:USERPROFILE\.cargo\bin;$env:PATH"
```

Useful commands:

```powershell
npm run build:about
npm run dev
npm run dev:about
npm run test:about
npm run test:rust
cargo check --manifest-path src-tauri\Cargo.toml
npm run build
```

Use `npm run dev:about` when checking About window layout. It shows the About window at startup. Do not rely on manually launching `target\debug\browser-switcher.exe` during `tauri dev` to trigger singleton behavior; that caused a WebView error page during development.

## Version And Release Rules

- Bump the version before creating a build intended for install or GitHub distribution.
- Merging to `main` runs the GitHub Actions release workflow.
- The release workflow publishes MSI/EXE assets only when `v<package.json version>` does not already exist.
- If a `main` merge should publish a new installer, bump the version in the same branch before merging.
- Keep version values aligned in:
  - `package.json`
  - `package-lock.json`
  - `src-tauri/Cargo.toml`
  - `src-tauri/Cargo.lock`
  - `src-tauri/tauri.conf.json`
- Regenerate About assets after version changes:

```powershell
npm run build:about
```

- `dist/index.html` should show the generated version from `package.json`.
- Windows installer/update behavior and icon caches may not refresh reliably if a new installer reuses the same version.
- Before testing an installer, fully quit any running `browser-switcher.exe` process.

## About Window Rules

- The OK button must hide the Tauri window with `getCurrentWindow().hide()`.
- Do not use `window.close()` for the OK button.
- Rust prevents close requests so closing the About window hides it instead of exiting the tray app.
- Using `window.close()` from the WebView can leave a blank or WebView error surface.

## Verification Expectations

Before claiming a change is complete, run the smallest relevant checks. For broad changes, run:

```powershell
npm run test:about
npm run test:rust
cargo check --manifest-path src-tauri\Cargo.toml
npm run build
```

For documentation-only changes, at least inspect `git diff` and verify links/commands are still accurate.

## Documentation Rules

- Keep `README.md` focused on users and GitHub visitors.
- Keep `AGENTS.md` focused on development rules and agent workflow.
- Keep `docs/development-summary.md` focused on current project context, dated decisions, caveats, and release notes.
- Do not turn `docs/development-summary.md` into a line-by-line changelog.
- Update `docs/development-summary.md` when behavior, build/release process, testing, or important Windows/Tauri assumptions change.
